Triggers — Webhook + Polling + Resume

Webhook (Port 8089)

Receives JIRA issue_created + issue_updated events. Rate limited (30/min per IP). HMAC signature validation. Instant response to new tickets.

Polling (Every 3 min)

JQL query: statusCategory in ("To Do","In Progress") + Bug, CR, P0 Alert. Catches anything webhook missed. Every 5th cycle scans waiting tickets for updates.

Resume on Update

When a ticket in WAITING_FOR_INFO or AWAITING_APPROVAL is updated, pipeline resumes from blocked stage. Works via both webhook and proactive polling.

Dedup (2 layers)

In-memory: shared known_keys set across poller + webhook.
JIRA-level: psv-im-processing label filter in JQL. Survives restart.

JIRA Status Transitions

Open → Under review → In Dev → Dev Fixed → Ready For QA → Closed. Also: Needs More Info (waiting), escalation labels.

Investigation Accuracy — 8 Data Sources

Ticket Comments

Parsed for JSON responses, HTTP status codes, error patterns. IDs extracted from comments too.

Linked Issues

Duplicates (0.95 score), relates-to, blockers. Checked before KB search.

Attachments

Downloaded + parsed: HAR files (requests/responses), log files (error lines), JSON (API errors).

Capillary Debug APIs

error-events, block-stats, route-events, jobs. Called based on RCA data_requests.

Cross-Project KB

Searches PSV + CAP + INTG for resolved tickets. Same bug fixed by another team = instant match.

BRD/SDD Context

Confluence SA + AG spaces. Brand aliases expanded. Relevant sections scored by keyword overlap.

Dataflow Versions

Version history fetched when dataflow_id present. Detects regression (recent deploy broke it).

EMF Logs (stub)

Configurable EMF_API_URL. Searches by request ID. Gracefully disabled when unconfigured.

5-Factor Confidence Breakdown

doc_quality (20%) + symptom_clarity (25%) + evidence_strength (30%) + pattern_match (15%) + data_completeness (10%) = weighted overall score. Shows WHY confidence is high or low.

Security — 6 Hardening Measures

Injection Prevention

CQL: _escape_cql() escapes quotes in Confluence queries.
JQL: Keywords filtered to alphanumeric, component escaped.
Prevents query manipulation from ticket data.

Path Traversal

Brand names sanitized to [a-zA-Z0-9_-] before use in file paths. realpath() validates no directory escape from state/ base.

Webhook Auth

HMAC-SHA256 signature validation when WEBHOOK_SECRET configured. Per-IP rate limiting (30 req/min). Client IP logged on failures.

Error Data Leak

Exception details logged locally only. JIRA comments get generic "check agent logs" message. No stack traces, connection strings, or internal paths exposed.

API Validation

All Capillary API IDs validated as alphanumeric before URL interpolation. Size params clamped to prevent abuse.

Secrets Management

.env gitignored. Never committed. .env.example has placeholders only. All credentials load via pydantic-settings.

IM Chatbot Integration

Agent delegates API calls to the IM Chatbot (which has per-brand credentials) instead of managing secrets itself.

Agent (Brain)

Runs full pipeline (Stages 0-5)
Identifies which APIs to call
Posts structured JIRA comment:
"IM Chatbot — please execute:
1. GET /v2/customers/{id}/statusLog
2. POST /searchEvaluationLog"
Sets WAITING_FOR_INFO, pauses.

Chatbot (Hands)

Has per-brand OAuth credentials
Reads the agent's JIRA comment
Executes API calls with brand auth
Posts JSON responses as comments
Agent resumes, extracts JSON evidence
Re-runs Stage 5 with real data.

No credential duplication Audit trail in JIRA comments Works with any brand Merge planned for shared backend

Deployment — 3 Run Modes

Polling + Webhook

python main.py

Default mode. Polls every 3 min + webhook on :8089 (if WEBHOOK_ENABLED=true). SLA monitor runs in background.

Webhook Only

python main.py --webhook-only

No polling. Pure event-driven. Instant response. Ideal for production with JIRA webhook configured.

Single Ticket

python main.py --investigate PSV-12345

Analyze one ticket. Shows results first, asks before posting to JIRA. Safe for testing.
