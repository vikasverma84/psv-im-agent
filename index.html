PSV-IM Agent

Autonomous hub for PSV incident management — polls Jira, validates against BRD/SDD, classifies concerns, routes to team skills, and resolves bugs with human approval at the gate.

Standalone — No Chatbot 10 AI Agents Pluggable Team Skills Component-Based Routing Atlassian + Capillary MCP
Vikas Verma • IM Team • April 2026

The Vision

"The bot should self-resolve as much as possible. TAMs/IMs see tickets getting resolved. SA/PM/Dev involvement only kicks in when absolutely necessary. Each team plugs in their own skill — the hub just routes and orchestrates."

Hub + Team Skills

The agent owns the pipeline. Each team (Integration, WebApp, Mobile, Config) attaches their own skill for domain-specific investigation and fixes. No monolith.

BRD/SDD Validation

Every ticket is validated against design docs before investigation. Bug vs SDD Gap vs BRD Gap vs CR — only bugs enter the fix pipeline. Gaps escalate.

Fully Standalone

No chatbot dependency. Talks directly to Atlassian MCP (Jira + Confluence) and Capillary MCP (platform APIs). All state is local files.

BRD/SDD Validation — The Gate

Every ticket is validated against design documents before entering the investigation pipeline. Only confirmed bugs proceed.

How It Works

SDDBRDClassificationAction
CoveredCoveredBugInvestigate + Fix
MissingCoveredSDD GapEscalate to SA
MissingMissingBRD GapEscalate to PM
Enhancement languageCRCR workflow

BRD/SDD Sources

1. Local cache (7-day TTL)
2. Atlassian MCP → Confluence SA + AG spaces
3. Jira ticket links / description URLs
4. Broad Atlassian search (last resort)

Bug

SDD says X should happen, but Y is happening. Behavior contradicts design. → Full pipeline runs.

SDD Gap

BRD requires it, but SDD has no design for this scenario. → SA must add missing design.

BRD Gap

Neither SDD nor BRD covers this. Undocumented requirement. → PM must evaluate scope.

Change Request

User wants different behavior than designed. Not a failure. → Route to CR workflow.

No docs available? Defaults to Bug with low confidence (0.3). Better to investigate than miss a real bug.

Skill Interface Contract

Hub provides:
• Parsed ticket (brand, component, org ID, dataflow ID)
• BRD/SDD context + relevant sections
• 5-Whys RCA hypothesis + data requests
• KB matches from past resolutions
• Brand config (cluster URL, Connect+ URL)

Team skill returns:
• Investigation findings + evidence
• Fix proposal (DEMO only, reversible)
• Test results (smoke + regression)

Each team creates ~/.claude/skills/{skill-id}/ with a /investigate command. Minimal skill: ~50 lines.
